# CV Template

Bilingual one-page CV with built-in PDF export. Copy this folder, swap content, push.

## Files

| File | Purpose |
| --- | --- |
| `index.html` | English version (also served as `/`) |
| `sv.html` | Swedish version |
| `avatar.jpeg` | Portrait — rendered as a 42×42px circle via `background-image` |
| `CNAME` | Custom domain for GitHub Pages |
| `CV-TEMPLATE.md` | This document |

There is no build step. The two HTML files are self-contained: CSS inline, Google Fonts via CDN, one tiny script for the footer date and the Save-as-PDF button.

## Page anatomy

In order, inside `<main class="page">`:

1. **Topbar** — `↓ Save as PDF` button (calls `window.print()`), EN/SV switch.
2. **Masthead** — avatar circle, name, two-line role (`.role-main` + `.role-sub`), and right-aligned contact block (email, phone, LinkedIn, site, location).
3. **Strip** — 5–6 availability `.chip` pills. The first one carries `.dot` for a small green status dot. Drop the `pdf-hide` class on any chip you want to skip in the PDF.
4. **Hero** — short pitch with one bold phrase wrapped in `<b>`.
5. **Hero-sub** — supporting paragraph.
6. **Three things I bring to the table** — three `.pillar` cards, each with `.num` eyebrow, `h3`, and one paragraph.
7. **How to hire me** — three `.pkg` cards: `.tag`, `h3`, `.dur`, intro paragraph, `<ul>` of 4 bullets, dashed rule, `.outcome` line.
8. **Selected clients** — `.brands` list, one `<span>` per name, rendered pipe-separated.
9. **Experience** — `.exp` grid: 12-column with year (col 1–3) and role/desc (col 4–12) per row.
10. **Tech stack + Education** — `.twocol` grid with two `.section`s side by side.
11. **CTA** — forest box with headline, one-liner, and two buttons (email + LinkedIn). Hidden in print.
12. **Footer** — name · location · "Updated <month> <year>" (auto-generated by `<script>` at the bottom).

## Content slots

For each language file, supply:

```
Header
  name, role main, role sub
  email, phone, LinkedIn URL, site URL, location

Chips (6 short labels)
  availability (with .dot)
  cadence (e.g. "1–3 days / week")
  location (optional — hide with .pdf-hide if duplicated in contact)
  languages
  legal / invoicing note
  hosting / data note

Hero
  one-sentence pitch with one bold phrase

Hero-sub
  4–6 lines of context

Three pillars (×3)
  num (e.g. "01 · ARCHITECTURE & SYSTEMS")
  h3 (headline statement)
  description (2–3 sentences)

Three packages (×3)
  tag ("Package 01")
  h3 (offering name)
  duration line
  intro sentence
  4 bullets
  outcome line

Selected clients
  flat list of brand names

Experience
  array of { years, role title, company, description }

Tech stack
  array of { label, content }

Education & languages
  list of { qualification + school, year / details }
  one entry for Languages, one for Publication (optional)

CTA
  headline, one-liner, two button labels

Footer
  name · location · Updated (date auto)
```

## Print / PDF

The `@media print` block at the bottom of the `<style>` block controls the PDF render. Key rules:

- `@page { size: A4; margin: 18mm 20mm; }`
- `print-color-adjust: exact` on all elements so accent colours survive
- White page background, paper tint dropped
- `.topbar` and `.cta` are hidden
- `.chip` strip flattens to inline `·` separated text
- `.pkg` cards compact to ~10pt h3 / 8pt body / 7.5pt bullets, with `white-space: nowrap` on h3
- `.avatar` uses `clip-path: circle(50%)` and drops the shadow so it prints as a clean disc
- `.pdf-hide` → `display: none`
- `.pdf-break` → `break-before: page`

Page breaks (controlled by `.pdf-break` on the relevant elements):

- Before **Selected clients** `<section>` → page 2 starts here
- Before **Tech stack** `.twocol` → final page starts here

To get a tighter 2-page CV: remove `pdf-break` from the `.twocol`. To get a roomier 3-page CV: also add `pdf-break` to the **How to hire me** section.

The PDF button is a plain `<button>` with `onclick="window.print()"`, styled as a small pill in the topbar.

## How to customise for a new person

1. Copy the repo to a new GitHub project.
2. Replace `avatar.jpeg` with the person's portrait. Square works best. If the face sits off-centre, tweak `background-position` on `.avatar` (default `32% 38%`).
3. In `index.html`, replace each content slot listed above. The HTML structure is stable — just swap the visible text.
4. Mirror everything in `sv.html` (or rename / drop it for monolingual sites — then also remove the `.locale` block).
5. Update `CNAME` to the new domain. Add a DNS CNAME record pointing the chosen host to `<user>.github.io`.
6. Push to `main`. In repo Settings → Pages, set source to `main` / root. Wait a minute for the build; HTTPS auto-provisions after DNS validates.

## Localisation pattern

There is no JS-driven i18n. Each language is its own HTML file:

- `index.html` carries `lang="en"`, the EN copy, and `class="active"` on the EN link.
- `sv.html` carries `lang="sv"`, the SV copy, and `class="active"` on the SV link.

To add a third language: copy `sv.html` to e.g. `de.html`, translate, then add a third `<a>` to the `.locale` nav in every file.

## Things to watch when editing

- Long titles in `.pkg h3` may overflow at print (we use `white-space: nowrap`). Shorten the title or shrink that one h3 if it clips.
- The `.exp` year column is 120px — long year ranges may need a wider column.
- Anything you want suppressed in the PDF can be tagged with `pdf-hide`.
- Anything you want to force onto a new printed page can be tagged with `pdf-break`.
- The mint left stripe is on `body { box-shadow }` — remove that line if you don't want it.

## Browser support

Targets evergreen Chrome / Safari / Firefox. PDF export works best in Chromium-based browsers (Chrome, Edge, Arc); Safari respects the print rules but may render colour adjustment slightly differently. Firefox's print engine is fine but doesn't yet honour `clip-path` on every render — the avatar shadow fix uses `clip-path: circle(50%)` plus a `box-shadow: none` reset as a fallback.
